# Testing

Puzzle Buddy uses XCTest for unit tests and XCUITest for UI automation. CI runs the full suite on every push and pull request.

## Test targets

| Target | Path | Purpose |
|--------|------|---------|
| **Puzzle BuddyTests** | `Puzzle BuddyTests/` | Unit tests — models, serialization, labels |
| **Puzzle BuddyUITests** | `Puzzle BuddyUITests/` | UI smoke and launch tests |

Both targets are wired in `project.yml` under the **Puzzle Buddy** scheme's `testTargets`.

## Running tests

### Xcode

⌘U with scheme **Puzzle Buddy** selected.

### Command line (CI parity)

```bash
xcodegen generate

xcodebuild build-for-testing \
  -project "Puzzle Buddy.xcodeproj" \
  -scheme "Puzzle Buddy" \
  -destination "platform=iOS Simulator,name=iPhone 16" \
  -derivedDataPath DerivedData \
  CODE_SIGN_IDENTITY=- \
  CODE_SIGNING_REQUIRED=NO \
  CODE_SIGNING_ALLOWED=NO

Scripts/ci/run-tests.sh "platform=iOS Simulator,name=iPhone 16"
```

Results bundle: `TestResults.xcresult`  
Log: `xcodebuild-test.log`

### Single test class

```bash
xcodebuild test-without-building \
  -project "Puzzle Buddy.xcodeproj" \
  -scheme "Puzzle Buddy" \
  -destination "platform=iOS Simulator,name=iPhone 16" \
  -only-testing:Puzzle_BuddyTests/PuzzleSerializationTests \
  -derivedDataPath DerivedData
```

## Unit tests

### `PuzzleSerializationTests`

Covers puzzle domain logic without Firebase:

- `PuzzleTime` formatting and parsing (`toName()`, `toMin()`, init from string)
- `getDataFields()` contains expected keys and values
- `Rating` and `Status` enum coverage
- `fromData(_:)` round-trip for future Firestore sync

### `PuzzlePersistenceTests`

Covers SwiftData persistence:

- `PuzzleRecord` round-trip from `Puzzle`
- `PuzzleStore` save and reload across store instances

When changing `Puzzle` or `PuzzleRecord` fields, extend these tests first.

### `AccessibilityLabelTests`

Validates accessibility contracts (labels/identifiers) for critical controls. Update when adding or renaming `A11yID` constants.

### `Puzzle_BuddyTests`

App-level tests including analytics allowlist mapping and `ProductService.isLoginEnabled` default (`false` for 1.0).

### Writing new unit tests

```swift
import XCTest
@testable import Puzzle_Buddy

final class MyFeatureTests: XCTestCase {
    func testExample() {
        let puzzle = Puzzle.fixture(name: "Test", pieces: 100)
        XCTAssertEqual(puzzle.name, "Test")
    }
}
```

Use `@testable import Puzzle_Buddy` to access internal types. Prefer fixtures in `Puzzle` extensions over large inline setup.

## UI tests

### `Puzzle_BuddyUITests`

Functional UI flows using accessibility identifiers from `A11yID`:

- **Default launch (1.0):** puzzle list, add button, settings — no login screen
- **Login tests:** launch with `-enable_login` via `UITestLaunch.loginArguments`
- `login_email_field`, `login_password_field`, `login_submit_button`
- `puzzle_list`, `add_puzzle_button`
- `puzzles_tab`, `settings_tab`, `settings_sign_out_button` (login enabled only)

### Launch arguments

| Argument | Purpose |
|----------|---------|
| `-disable_firebase_analytics` | Skip Analytics in tests |
| `-enable_login` | Show login UI (`ProductService`) |
| `-ui_testing_bypass_auth` | Skip login gate when login is enabled |
| `-ui_testing_seed_puzzles` | Insert fixture puzzles into SwiftData |

### `Puzzle_BuddyUITestsLaunchTests`

Launch performance / screenshot baseline tests generated by Xcode.

### Locating elements

```swift
let email = app.textFields[A11yID.loginEmailField]
XCTAssertTrue(email.waitForExistence(timeout: 5))
email.tap()
email.typeText("test@example.com")
```

Always use `A11yID` constants — avoid brittle label-only queries when an identifier exists.

### UI tests and Firebase

**1.0 default:** UI tests launch into the main puzzle UI with SwiftData — no sign-in required.

Login-screen tests pass `-enable_login`. Seeded puzzle tests use `-ui_testing_seed_puzzles` (and optionally `-ui_testing_bypass_auth` when login is enabled).

CI uses `GoogleService-Info.plist.example` — Firebase does not fully initialize, so tests should not depend on live sync unless you add test-specific configuration.

### Accessibility audits

See [wcag.md](wcag.md) for WCAG 2.1 AA criteria and manual VoiceOver/Dynamic Type procedures.

Planned `XCUIAccessibilityAudit` tests should run on login, puzzle list, puzzle form, and settings. VoiceOver scripts live in [accessibility/wcag-2.1-aa/voiceover-scripts/](../accessibility/wcag-2.1-aa/voiceover-scripts/).

## CI integration

Workflow: `.github/workflows/ci.yml`

1. Copy example Firebase plist
2. `xcodegen generate`
3. Resolve packages
4. `build-for-testing`
5. `Scripts/ci/run-tests.sh`
6. Upload `TestResults.xcresult` on completion (even on failure)

Parallel testing is disabled (`CI_PARALLEL_TESTING: NO`) for predictable simulator behavior.

## Coverage expectations for contributors

| Change | Minimum testing |
|--------|-----------------|
| New `Puzzle` / `PuzzleRecord` field | Unit test in `PuzzleSerializationTests` and/or `PuzzlePersistenceTests` |
| New primary button / field | `A11yID` + label test or UI test query |
| Auth flow change | Manual test plan with `-enable_login` + unit tests where pure logic exists |
| `ProductService` flag change | Update `Puzzle_BuddyTests` and affected UI tests |
| Firestore path or rules | Manual verify when login ships + consider Firebase emulator (future) |

## Debugging failures

### Simulator not found

```bash
xcrun simctl list devices available
```

Update destination to an installed runtime.

### Code signing errors

CI uses `CODE_SIGNING_ALLOWED=NO`. Match locally for CLI builds or sign in Xcode for device runs.

### Flaky UI tests

- Increase `waitForExistence(timeout:)`
- Reset simulator state: Device → Erase All Content and Settings
- Disable animations: Simulator → Settings → Accessibility → Reduce Motion

### Reading xcresult

Open in Xcode: Window → Organizer → Reports, or:

```bash
open TestResults.xcresult
```
