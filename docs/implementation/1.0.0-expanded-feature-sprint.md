## Phase 1 — Schema & domain (Features B, C, D, E)

### B — Purchase price

| Layer | Field | Type | Notes |
|-------|-------|------|-------|
| `PuzzleRecord` | `purchasePrice` | `Double?` | Store as decimal; nil = unknown |
| `PuzzleRecord` | `purchaseCurrencyCode` | `String?` | Default `Locale.current.currency?.identifier ?? "USD"` on save when price set |
| `Puzzle` | matching `@Published` props | | Format with `FormatStyle.Currency` in UI |

**Validation:** `purchasePrice >= 0`, max 999_999.99; clear price clears currency.

**Stats (Phase 5):** Optional “Collection value” card on Stats tab (sum of known prices) — ship if trivial, else defer to 1.0.1.

### C — Shape, dimensions, cut type

Add to `PuzzleMetadataEnums.swift`:

| Enum | Cases (initial) |
|------|-----------------|
| `PuzzleShape` | none, rectangular, square, round, irregular |
| `PuzzleCutType` | none, ribbon, grid, random, unknown |

| Field | Type | UI |
|-------|------|-----|
| `puzzleShape` | enum raw string on record | Picker on form; detail row if not `.none` |
| `cutType` | enum raw string | Picker on form; detail row if not `.none` |
| `dimensionsText` | `String?` | TextField, e.g. `68 × 49 cm` or `27 × 19 in`; max 80 chars |

Detail-only display is fine for dimensions if you prefer form-only entry — but **form + detail** matches existing metadata pattern.

IPDb CSV: map if columns exist; otherwise omit from export columns until IPDb format confirmed.

### D — Multi-photo gallery

New SwiftData model:

```swift
@Model
final class PuzzlePhotoRecord {
    @Attribute(.unique) var id: UUID
    var puzzleID: UUID          // parent puzzle id (denormalized for queries)
    var sortOrder: Int          // 0 = cover
    var caption: String?        // "Box", "Progress", "Completed"
    @Attribute(.externalStorage) var imageData: Data?
    var createdAt: Date
}
```

**Migration strategy:**

1. Add `PuzzlePhotoRecord` to `Schema([PuzzleRecord.self, PuzzlePhotoRecord.self])`.
2. On first launch after update: if puzzle has legacy `imageData` and zero photos, create one `PuzzlePhotoRecord` at `sortOrder: 0` (one-time migration helper in `PuzzleStore` or app launch).
3. Keep `PuzzleRecord.imageData` **deprecated** for one release — read fallback only; writes go to photos. Remove in 1.1 cleanup.

**Limits:** Max **5** photos per puzzle (`PuzzlePhotoLimits.maxCount`).

**Cover rule:** `sortOrder == 0` is cover; list cells and collage use cover photo.

### E — Redo + completion tracking

New SwiftData model:

```swift
@Model
final class PuzzleCompletionRecord {
    @Attribute(.unique) var id: UUID
    var puzzleID: UUID
    var completionNumber: Int     // 1-based, sequential per puzzle
    var startedAt: Date?
    var completedAt: Date
    var timeSpentHours: Int?
    var timeSpentMinutes: Int?
    var rating: Double?           // snapshot at completion; optional
}
```

**Denormalized on puzzle:** `timesCompleted: Int` — equals `completions.count`; updated on each completion event for fast list/stats.

**Rules:**

1. **First time → Completed:** create completion #1 with current `startDate`, `completionDate`, time spent, rating.
2. **Redo:** Detail toolbar action **“Puzzle again”** (completed puzzles only):
   - Append nothing yet; set status → `In-Progress`, `progressPercent` → 0, `startDate` → now, clear `completionDate` to now (placeholder until next complete).
3. **Subsequent → Completed:** create completion #N+1; increment `timesCompleted`.
4. **Detail:** Show `Completed N times` when `timesCompleted > 1`; expandable list of past completion dates (+ time if set).

**Not in v1 of this feature:** separate photo galleries per attempt (defer); timer integration (manual time only).
