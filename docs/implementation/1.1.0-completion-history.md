# 1.1.0 ‚Äî Completion history edit & delete

**Branch:** `release/1.1.0`  
**Target:** 1.1.0  
**Builds on:** 1.0.0 completion records (`PuzzleCompletionRecord`, redo, read-only detail rows)

## Problem

Collectors can accidentally mark a puzzle **Completed**, or want to fix/remove an entry in their finish history (wrong date, duplicate tap, test entry). Today:

- Completing appends a `PuzzleCompletionRecord` and bumps `timesCompleted`.
- Detail shows **Times completed** plus one row per completion (read-only).
- **Puzzle again** starts a new cycle without deleting history.
- There is **no** way to delete or edit a completion log.

## Goals

1. **Review history** ‚Äî clear list of completions (newest first) with date, optional time spent, optional rating snapshot.
2. **Edit a completion** ‚Äî change completed date (and optional time-spent snapshot on that record).
3. **Remove a completion** ‚Äî delete one log; fix accidental ‚Äúmark complete‚Äù.
4. **Stay consistent** ‚Äî `timesCompleted`, stats, milestones, and puzzle status/date fields stay in sync after edits.

## Non-goals (1.1.0)

- Per-completion photo galleries
- In-app timer per attempt
- Bulk edit across puzzles

## Data model

No schema change. Use existing `PuzzleCompletionRecord` fields.

After any mutation, **reconcile** the parent `Puzzle`:

| Field | Rule |
|-------|------|
| `timesCompleted` | `completions.count` |
| `completionNumber` | Renumber 1‚Ä¶N in chronological `completedAt` order |
| `completionDate` | Newest completion‚Äôs `completedAt` (or keep sensible default if none) |
| `status` | If `completions` empty and was `.completed` ‚Üí prompt: revert to **In progress** (preserve progress %) or **To-Do** |
| Stats / milestones | Recompute from store after save (existing `CollectionStats.compute`) |

## Store API (`PuzzleStore`)

```swift
func deleteCompletion(puzzleID: UUID, completionID: UUID) throws
func updateCompletion(puzzleID: UUID, completion: PuzzleCompletion) throws
private func reconcileCompletions(for puzzleID: UUID) throws
```

- `deleteCompletion`: remove record, renumber, reconcile puzzle + save.
- `updateCompletion`: replace matching record by `id`, re-sort/renumber if dates changed, reconcile.
- Do **not** append a new completion when only editing/deleting existing rows.

## UX

### Detail ‚Äî Completion history section

Replace inline read-only rows with a **Completion history** group (when `timesCompleted > 0` or `completions` non-empty):

- Header: ‚ÄúCompletion history‚Äù + count badge.
- Each row: `#N ¬∑ Jan 12, 2026` (+ time spent if present).
- Swipe actions or context menu: **Edit**, **Delete**.
- Tap row ‚Üí edit sheet (date picker; optional hours/minutes for that attempt).

### Delete confirmation

- Single completion, puzzle still **Completed**:  
  ‚ÄúRemove this completion? You can set the puzzle back to In progress or To-Do.‚Äù  
  Actions: **Remove & keep Completed** (only if other completions remain ‚Äî otherwise force status pick), **Remove & set In progress**, **Remove & set To-Do**, Cancel.
- Multiple completions:  
  ‚ÄúRemove completion #2 from Jan 12, 2026? Your other finishes stay in history.‚Äù

### Accidental complete (status ‚Üí Completed)

Optional fast path on detail when user just completed (same session or within N minutes ‚Äî **defer** if complex):

- Banner or action: **Undo completion** ‚Üí deletes the completion record just appended and restores previous status/progress.

Minimum for 1.1.0: delete via history list is enough; undo banner is nice-to-have if cheap.

### Edit mode

Completion history edits work in **view** mode (dedicated sheet), not only puzzle edit form ‚Äî avoids coupling to full puzzle edit.
